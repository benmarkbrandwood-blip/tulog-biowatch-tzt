# ADS1293 ECG Front-End — Driver Reference

Board: TZT ESP32-2432S024C  
Breakout: Protocentral ADS1293 breakout (same as the ProtoC Arduino library)

---

## GPIO Connections

The ADS1293 shares the SPI2/HSPI bus with the ILI9341 display and XPT2046 touch
controller. No bus-initialisation changes are needed; `hal_display_init()` owns the
bus and `hal_ads1293_init()` calls `hal_spi_dev_add()` to register a second device.

| Signal | ESP32 GPIO | Direction | Notes |
|--------|-----------|-----------|-------|
| SCLK   | GPIO14    | Out       | Shared with display; never use as ADC |
| MOSI   | GPIO13    | Out       | Shared |
| MISO   | GPIO12    | In        | Shared |
| CS     | GPIO16    | Out       | Active-low; R20 (LED B resistor) removed to free this pin |
| DRDY   | GPIO4     | In        | Active-low; **no external pull-up** — R17 (RGB G, 1 kΩ) removed with LED |

> **CS pin**: R20 (the 1 kΩ series resistor on the RGB LED Blue cathode) was
> removed from the TZT board to free GPIO16.
>
> **DRDY pin (GPIO4)**: The RGB LED and all three LED series resistors (R17, R16,
> R20) have been physically removed from the board. R17 was the 1 kΩ pull-up on
> GPIO4; it is no longer present. Only the ESP32 internal ~45 kΩ pull-up is active.
> The ADS1293 DRDYB is open-drain; with only a 45 kΩ pull-up the pin may not assert
> reliably. If DRDY never goes LOW, reads are performed unconditionally at 100 Hz —
> DATA_STATUS register confirms ECG data is available regardless of the pin state.
> **To restore reliable DRDY**: add a 4.7 kΩ–10 kΩ pull-up resistor from GPIO4 to
> 3.3 V, or wire the ADS1293 breakout's IOREF/VCC to 3.3 V if it has an on-board
> pull-up.

## SPI Protocol

| Parameter | Value |
|-----------|-------|
| Bus | SPI2 / HSPI |
| Clock speed | 4 MHz (max 20 MHz) |
| **Mode** | **0 (CPOL=0, CPHA=0)** — confirmed by Protocentral Arduino reference |
| Bit order | MSB first |
| CS polarity | Active-low, held for full transaction |

**Command byte encoding:**
```
Write: bit7=0, bits[6:0]=register address
Read:  bit7=1, bits[6:0]=register address
```

In our driver (`hal_ads1293_regs.h`):
```c
#define ADS1293_CMD_WRITE(addr)  ((uint8_t)((addr) & 0x7F))
#define ADS1293_CMD_READ(addr)   ((uint8_t)(0x80u | ((addr) & 0x7F)))
```

Each transaction is a 16-bit (2-byte) exchange for single-register access, or a
multi-byte burst for DATA_LOOP reads.

## Electrode Wiring (3-lead Einthoven)

| Electrode | Colour convention | ADS1293 input |
|-----------|------------------|---------------|
| RA (Right Arm) | Red   | IN1 |
| LA (Left Arm)  | Yellow/Black | IN2 |
| LL (Left Leg)  | Green | IN3 |
| RL (Right Leg) | Black/Green | IN4 (RLD output) |

Derived leads:
- **Lead I**   = LA − RA = IN2 − IN1 → FLEX_CH1_CN = 0x11
- **Lead II**  = LL − RA = IN3 − IN1 → FLEX_CH2_CN = 0x19
- **Lead III** = LL − LA = IN3 − IN2 → FLEX_CH3_CN = 0x13

## Initialisation Register Sequence

Performed in `hal_ads1293_init()` (`main/hal/hal_ads1293.c`).

| Register | Address | Value | Description |
|----------|---------|-------|-------------|
| CONFIG | 0x00 | 0x00 | Stop conversion first |
| REVID | 0x40 | — | Read; verify = 0x01 |
| FLEX_CH1_CN | 0x01 | 0x11 | Lead I: INP=IN2, INN=IN1 |
| FLEX_CH2_CN | 0x02 | 0x19 | Lead II: INP=IN3, INN=IN1 |
| FLEX_CH3_CN | 0x03 | 0x13 | Lead III: INP=IN3, INN=IN2 |
| CMDET_EN | 0x0A | 0x07 | CM detection on IN1, IN2, IN3 |
| RLD_CN | 0x0C | 0x04 | RLD output → IN4 |
| OSC_CN | 0x12 | 0x04 | Bit 2 = CLKFED: feed RC oscillator to digital section |
| AFE_SHDN_CN | 0x14 | **0x00** | **All AFE channels active** (0x24 shuts down channels → DRDY never fires) |
| R2_RATE | 0x21 | 0x02 | R2 = 5 |
| R3_RATE_CH1 | 0x22 | 0x02 | R3 = 6 → ODR ≈ 853 SPS |
| R3_RATE_CH2 | 0x23 | 0x02 | R3 = 6 |
| R3_RATE_CH3 | 0x24 | 0x02 | R3 = 6 |
| DRDYB_SRC | 0x27 | 0x08 | CH1 ECG drives DRDYB pin |
| CH_CNFG | 0x2F | 0x70 | E1+E2+E3 enabled in DATA_LOOP |
| CONFIG | 0x00 | 0x01 | Start conversion |

**ODR formula:** `102400 / (R1 × R2 × R3) = 102400 / (4 × 5 × 6) ≈ 853 SPS`

> **Critical bugs fixed vs earlier code:**
> 1. SPI mode was `1` (CPHA=1) — should be `0` (CPHA=0). Arduino reference uses
>    `SPI_MODE0`; the ADS1293 samples SDI on the rising SCLK edge.
> 2. `AFE_SHDN_CN = 0x24` shut down AFE blocks — DRDY never asserted. Fix: `0x00`.
> 3. `ADS1293_AFE_RES` was mapped to address `0x11` (actually REF_CN). Correct
>    address is `0x13` (fixed in `hal_ads1293_regs.h`).

## DRDY Polling and Data Read

DRDY is active-low on GPIO4, but the external pull-up (R17) has been removed.
The acquisition service reads unconditionally every 100 Hz tick.

```
ecg_sampler_task (100 Hz)
  → ecg_adc_read_raw()
      → svc_biosignal_acq_step_100hz()
          → hal_ads1293_read_ecg()   // unconditional — DRDY gate removed
```

The ADS1293 runs at ~853 SPS. Reading at 100 Hz (every ~8–9 samples the chip
has produced) is sufficient for the ECG pipeline. `hal_ads1293_data_ready()` still
exists and is correct; it is just not called as a gate.

> **Current status (2026-06-13):** All 13 init registers verified via write-readback.
> DATA_STATUS = 0xFC (ECG1/ECG2/ECG3 all report data ready). Physical DRDY GPIO4
> never goes LOW — suspected cause: no external pull-up after R17 removal. Reads are
> now unconditional; CH1/CH2/CH3 raw values logged every 100 steps (~1 Hz) to
> diagnose whether valid signals are returned.

## DATA_LOOP Read (3 channels, 10 bytes)

With `CH_CNFG = 0x70` (E1+E2+E3), a DATA_LOOP read returns 9 data bytes:
CH1[H,M,L] + CH2[H,M,L] + CH3[H,M,L].

```c
uint8_t tx[10] = { ADS1293_CMD_READ(ADS1293_DATA_LOOP), 0,0,0,0,0,0,0,0,0 };
uint8_t rx[10] = { 0 };
hal_spi_txrx(s_spi, tx, rx, 10);

// rx[0]  = don't-care (MISO during cmd byte)
// rx[1..3] = CH1 ECG MSB, mid, LSB  (Lead I)
// rx[4..6] = CH2 ECG MSB, mid, LSB  (Lead II)
// rx[7..9] = CH3 ECG MSB, mid, LSB  (Lead III)
```

**24-bit sign extension to int32_t:**
```c
*ch1_out = (int32_t)(((uint32_t)rx[1] << 24) |
                     ((uint32_t)rx[2] << 16) |
                     ((uint32_t)rx[3] <<  8)) >> 8;
```

The shift-by-24-then-right-shift-by-8 pattern performs arithmetic sign extension
without UB: the intermediate cast to `uint32_t` avoids signed overflow, and the
final `>> 8` on `int32_t` propagates the sign bit.

**Scale to 12-bit pipeline range (in ecg_adc_read_raw):**
```c
int raw = (int)(frame.ecg1_raw >> 12) + 2048;
// >> 12 maps ±2^23 → ±2048; +2048 re-centres to 0–4095 (baseline 2048)
```

## Signal Routing in Firmware

| ADS1293 channel | Scaled variable | Ring buffer | Display |
|----------------|----------------|-------------|---------|
| CH1 — Lead I   | `s_ecg_last_raw` | `s_ecg_raw[]` | ECG chart series 1 (white) |
| CH2 — Lead II  | `s_ecg2_scaled` | `s_fcg2_raw[]` | ECG chart series 2 (yellow) |
| CH3 — Lead III | `s_ecg3_scaled` | `s_ecg3_raw[]` | ECG chart series 3 (cyan) |

FCG2 tab on the record screen shows **simulated** data (`rec_fill_sim_plot`) until a
real FCG sensor is wired. The `s_fcg2_raw[]` ring buffer holds Lead II data and is
used exclusively by the ECG chart's second series.
