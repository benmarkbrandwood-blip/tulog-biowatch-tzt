# MAX30102 Pulse Oximeter & Heart-Rate Sensor — Driver Reference

Board: TZT ESP32-2432S024C (resistive touch variant)
Sensor: Maxim Integrated MAX30102 (common GY-MAX30102 breakout module)

---

## GPIO Connections

The MAX30102 uses the shared I²C0 bus alongside the MPU-6050/6500.

| Signal | ESP32 GPIO | Direction | Notes |
|--------|-----------|-----------|-------|
| SDA    | GPIO21    | Bidirectional | 10 kΩ external pull-up to 3.3 V required (or module on-board pull-up) |
| SCL    | GPIO22    | Out        | 10 kΩ external pull-up to 3.3 V required (or module on-board pull-up) |
| INT    | GPIO32    | In         | Not used in firmware — FIFO reads are unconditional |
| VCC    | 3.3 V     | —          | Module must be on 3.3 V rail |

> **Pull-up note**: Most GY-MAX30102 breakout modules include 4.7 kΩ on-board
> pull-ups on SDA and SCL, so no additional external pull-ups are needed when
> the module is used alone. When sharing the bus with a bare MPU-6050 module
> (no on-board pull-ups), ensure 10 kΩ pull-ups are present on the bus lines.

## Sensor Configuration

`hal_max30102_init()` ([main/hal/hal_max30102.c](../main/hal/hal_max30102.c)) applies
the following configuration after verifying Part ID and issuing a soft reset.

### Mode configuration

| Register | Address | Value | Effect |
|----------|---------|-------|--------|
| MODE_CONFIG | 0x09 | 0x40 | RESET bit — soft reset; clears all registers |
| MODE_CONFIG | 0x09 | 0x03 | SpO₂ mode — Red + IR LEDs both active |

### FIFO configuration

| Register | Address | Value | Effect |
|----------|---------|-------|--------|
| FIFO_CONFIG | 0x08 | 0x10 | No sample averaging; FIFO rollover enabled |
| FIFO_WR_PTR | 0x04 | 0x00 | Clear FIFO write pointer |
| OVF_COUNTER | 0x05 | 0x00 | Clear overflow counter |
| FIFO_RD_PTR | 0x06 | 0x00 | Clear FIFO read pointer |

### SpO₂ configuration

| Register | Address | Value | Bits | Effect |
|----------|---------|-------|------|--------|
| SPO2_CONFIG | 0x0A | 0x27 | [6:5] = 01 | ADC full-scale range: 4096 nA |
| | | | [4:2] = 001 | Sample rate: 100 SPS |
| | | | [1:0] = 11 | Pulse width: 411 µs → 18-bit ADC resolution |

At 411 µs pulse width, the effective ADC resolution is **18 bits** (unsigned,
range 0–262,143).

### LED currents

| Register | Address | Value | Effect |
|----------|---------|-------|--------|
| LED1_PA (Red) | 0x0C | 0x24 | 7.2 mA drive current |
| LED2_PA (IR)  | 0x0D | 0x24 | 7.2 mA drive current |

Current = register × 0.2 mA, so 0x24 = 36 × 0.2 = 7.2 mA.

## FIFO Read Procedure

`hal_max30102_read_fifo(int32_t *ir_out, int32_t *red_out)` performs the following
on every call from `svc_biosignal_acq_step_100hz()`:

1. Burst-read 3 bytes: `FIFO_WR_PTR` (0x04), `OVF_COUNTER` (0x05), `FIFO_RD_PTR`
   (0x06). These are contiguous registers so one I²C transaction reads all three.

2. Compute available samples: `n_avail = (wr_ptr - rd_ptr) & 0x1F`.

3. Return `ESP_ERR_NOT_FOUND` if `n_avail == 0` (no new samples since last call).

4. Cap drain at 4 samples to bound the I²C transaction to 24 bytes.

5. Burst-read `n_avail × 6` bytes from `FIFO_DATA` (0x07). The I²C address
   auto-increments through FIFO entries.

6. For each sample: extract Red (3 bytes) then IR (3 bytes); feed both to the
   SpO₂ accumulator; overwrite `*ir_out` and `*red_out` with each sample so the
   return values hold the **most recent** (freshest) sample.

**Byte layout per sample (SpO₂ mode):**

```
Byte 0: LED1 (Red) bits [17:16] in bits [1:0]; bits [7:2] unused
Byte 1: LED1 (Red) bits [15:8]
Byte 2: LED1 (Red) bits [7:0]
Byte 3: LED2 (IR)  bits [17:16] in bits [1:0]; bits [7:2] unused
Byte 4: LED2 (IR)  bits [15:8]
Byte 5: LED2 (IR)  bits [7:0]
```

**Extraction:**

```c
int32_t red = ((int32_t)(s[0] & 0x03) << 16) | ((int32_t)s[1] << 8) | s[2];
int32_t ir  = ((int32_t)(s[3] & 0x03) << 16) | ((int32_t)s[4] << 8) | s[5];
```

## SpO₂ Algorithm

The SpO₂ computation runs inside `spo2_accumulate()` using a **Welford online
variance** accumulator over a rolling 400-sample (~4 s) window. No sample history
is stored.

### Accumulation (per sample)

```
s_n++
dr = red[n] − red_mean
red_mean += dr / s_n
red_M2   += dr × (red[n] − red_mean)     ← Welford M2 update

di = ir[n] − ir_mean
ir_mean  += di / s_n
ir_M2    += di × (ir[n] − ir_mean)
```

### SpO₂ estimate (every 400 samples)

```
AC_red = √(red_M2 / n)         ← RMS AC amplitude ≈ std deviation
AC_ir  = √(ir_M2  / n)

R = (AC_red / DC_red) / (AC_ir / DC_ir)    where DC_red = red_mean, DC_ir = ir_mean

SpO₂ = −45.060 × R² + 30.354 × R + 94.845   (% saturation)
```

The quadratic calibration curve is derived from the Maxim Integrated reference
design (Application Note 6409). For physiological R values of 0.4–1.0 it
approximates the Beer-Lambert relationship between haemoglobin absorption ratios
and arterial oxygen saturation.

### Validity gates

- If `ir_mean < 5000` or `red_mean < 5000` ADC counts: no finger detected →
  SpO₂ = 0.
- If computed SpO₂ < 50 %: out of plausible range → SpO₂ = 0.
- If computed SpO₂ > 100 %: clamped to 100.
- After each 400-sample window, accumulators reset to zero for the next window.

SpO₂ is published via `hal_max30102_get_spo2()` and written to `rec_row_t.spo2`
(`uint8_t`, 0–100 %) in the CSV. A log is printed at each update:

```
I hal_max30102: SpO2 98% (R=0.512 DC_ir=145230.0 DC_red=98460.0)
```

## Firmware Integration

### Acquisition path

```
svc_biosignal_acq_step_100hz()    (100 Hz, from ecg_sampler_task)
    └── hal_max30102_read_fifo()
            ├── burst-read FIFO pointers + data (I²C)
            ├── spo2_accumulate() → internal window accumulator
            └── publish ppg_ir_raw, ppg_red_raw to biosignal_frame_t (mutex)

ecg_sampler_task
    └── read biosignal_frame_t (BIOSIG_VALID_PPG_IR)
            └── s_ppg_ir_latest → bandpass → foot detector → chart / CSV
```

### PPG source selection

At runtime in `ecg_sampler_task`:

```c
int32_t ppg_raw = (s_ppg_ir_latest > 0)
                  ? s_ppg_ir_latest
                  : (int32_t)ppg_sim_get_sample(expected_ms);
```

When the MAX30102 is online, `s_ppg_ir_latest` holds the most recent 18-bit IR
count and the simulation path is bypassed. The sample then flows through the same
`signal_bandpass_step()` (0.5–16 Hz), `ppg_det_update_sample()` (PAT foot
detector), ring buffer, chart display, and CSV — all amplitude-agnostic.

### CSV output

| Column | Type | Range | Description |
|--------|------|-------|-------------|
| `ppg` | int32 | 0–262 143 | IR channel from MAX30102 FIFO (18-bit); simulated if sensor absent |
| `spo2` | uint8 | 0–100 | SpO₂ % from Welford estimate; 0 if no finger or window < 400 samples |

SpO₂ updates approximately every 4 s; between updates the CSV column holds the
previous window's result (non-zero once a finger has been detected).

## Part ID Verification

`hal_max30102_init()` reads register 0xFF and verifies it equals `0x15` before
proceeding. If the Part ID is wrong (or the I²C read fails), init returns an error
and the PPG channel falls back to simulation.

```
I hal_max30102: Part ID 0x15 OK
I hal_max30102: Init OK — SpO2 mode, 100 SPS, 18-bit, LED 7.2 mA
I svc_biosig: MAX30102 online — PPG IR/Red + SpO2 valid
```

## Known Limitations

- **SpO₂ topbar label** — `s_lbl_rec_spo2` on the record screen currently shows
  `"--%"` and is not wired to the live SpO₂ value. The CSV column is correct.
  Wiring the label is a pending UI task.
- **INT pin (GPIO32)** — not used; FIFO reads are unconditional at 100 Hz. For
  power-critical operation, configure the INT output (register 0x02) to assert on
  PPG_RDY and gate reads from the ISR.
- **LED current tuning** — 7.2 mA works for index-finger contact. For wrist PPG
  (lower perfusion), increase LED current toward 20–25 mA (0x64–0x7D).
